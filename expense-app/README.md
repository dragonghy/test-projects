# Expense Tracker - 个人记账仪表盘

**在线演示**: https://expense-app-murex.vercel.app

**部署状态**: ✅ 已部署到 Vercel

## 产品概述

### 目标
为个人用户提供一个简洁、直观的日常开支记录和分析工具。用户可以快速记账、查看消费趋势、了解支出分布，从而更好地管理个人财务。

### 项目类型
- **技术栈**：纯前端 Web 应用（HTML + CSS + JavaScript，不使用框架）
- **部署方式**：Vercel 静态部署
- **代码目录**：`projects/expense-app/`
- **文件结构**：`index.html`, `style.css`, `app.js`
- **数据存储**：浏览器 localStorage

### 目标用户
想要简单记录和分析日常开支的个人用户。无需注册登录，打开即用。

---

## 用户交互设计

### 核心体验流程
1. 用户打开应用 → 看到当月仪表盘概览（总支出、笔数、日均）
2. 点击"添加支出" → 填写金额、选择分类、选择日期、输入备注 → 保存
3. 支出列表实时更新，按时间倒序显示
4. 切换月份查看历史数据
5. （M2）查看分类饼图和趋势柱状图，设置预算，搜索和导入导出数据

### 页面布局
- **桌面端**：双列布局，左侧为添加表单 + 图表区，右侧为支出列表
- **移动端**：单列布局，表单在上，列表在下

## Milestone 规划

### M1：基础记账功能

**目标**：实现完整的记账 CRUD 和月度概览，用户可以正常记账并查看支出。

#### 功能清单

| # | 功能 | 描述 |
|---|------|------|
| 1.1 | 添加支出 | 表单包含：金额（必填，数字）、分类（必填，下拉选择）、日期（必填，默认今天）、备注（选填，文本） |
| 1.2 | 预设分类 | 8 个预设分类：餐饮、交通、购物、娱乐、住房、医疗、教育、其他。每个分类有对应颜色和图标（可用 emoji） |
| 1.3 | 支出列表 | 按时间倒序排列，每条记录显示：金额、分类标签（彩色）、日期、备注。仅显示当前选中月份的数据 |
| 1.4 | 编辑支出 | 点击列表中的记录可编辑，复用添加表单 |
| 1.5 | 删除支出 | 每条记录有删除按钮，删除前需确认 |
| 1.6 | 月份筛选 | 仪表盘顶部显示当前月份，有上一月/下一月按钮切换 |
| 1.7 | 仪表盘概览 | 顶部卡片区显示当月三个指标：总支出、笔数、日均支出 |
| 1.8 | 数据持久化 | 所有数据保存在 localStorage，刷新页面数据不丢失 |
| 1.9 | 响应式设计 | 桌面端双列布局（≥768px），移动端单列布局（<768px） |
| 1.10 | UI 风格 | 现代简洁风格，浅色主题，圆角卡片，分类彩色标签，适当的间距和字体层级 |

#### M1 验收标准

- [ ] **AC-1.1** 添加支出：填写金额 100、选择"餐饮"、日期为今天、备注"午餐" → 点击保存 → 列表中出现该记录，仪表盘数据更新
- [ ] **AC-1.2** 金额验证：输入非数字或负数 → 提示错误，不允许保存
- [ ] **AC-1.3** 分类选择：下拉菜单显示 8 个预设分类，每个分类有对应颜色标识
- [ ] **AC-1.4** 支出列表：添加多条记录后，列表按时间倒序排列，最新的在最上面
- [ ] **AC-1.5** 编辑功能：点击一条记录 → 表单填充该记录数据 → 修改金额 → 保存 → 列表更新
- [ ] **AC-1.6** 删除功能：点击删除 → 弹出确认 → 确认后记录消失，仪表盘数据更新
- [ ] **AC-1.7** 月份切换：点击"上一月"/"下一月" → 列表和仪表盘数据切换到对应月份
- [ ] **AC-1.8** 仪表盘数据：当月有 3 笔支出（100+200+300=600）→ 总支出显示 600，笔数 3，日均 = 600/当月天数
- [ ] **AC-1.9** 数据持久化：添加数据后刷新页面 → 数据仍然存在
- [ ] **AC-1.10** 响应式：桌面端显示双列布局，缩小窗口到 <768px 时变为单列
- [ ] **AC-1.11** 空状态：没有数据时显示友好的空状态提示（如"还没有支出记录，开始记账吧"）

### M2：增强功能

**目标**：添加可视化图表、预算管理、搜索和数据导入导出功能，提升数据分析能力。

**前置条件**：M1 验收通过

#### 功能清单

| # | 功能 | 描述 |
|---|------|------|
| 2.1 | 分类饼图 | 使用 Canvas 绘制，显示当月各分类支出占比百分比，鼠标悬停显示具体金额 |
| 2.2 | 月度趋势柱状图 | 使用 Canvas 绘制，显示近 6 个月的支出总额对比，直观展示消费趋势 |
| 2.3 | 预算设置 | 用户可设置每月预算上限，仪表盘显示预算进度条（已用/总预算），超出预算时进度条变红 |
| 2.4 | 支出搜索 | 在列表上方添加搜索框，按备注关键字实时筛选支出记录 |
| 2.5 | CSV 导出 | 导出当前月份的支出数据为 CSV 文件下载，包含列：日期、分类、金额、备注 |
| 2.6 | CSV 导入 | 上传 CSV 文件导入支出数据，格式与导出一致，导入前显示预览和确认 |
| 2.7 | 深色/浅色主题切换 | 右上角添加主题切换按钮，切换深色/浅色模式，选择保存到 localStorage |

#### M2 验收标准

- [ ] **AC-2.1** 饼图：当月有"餐饮 300 + 交通 200 + 购物 500"时 → 饼图显示三个扇区，分别标注分类名和百分比（如"购物 50%"）
- [ ] **AC-2.2** 柱状图：近 6 个月有数据时 → 显示 6 根柱子，高度与金额成比例，每根柱子下方标注月份
- [ ] **AC-2.3** 预算设置：设置预算 1000 → 仪表盘显示进度条；当月支出 800 → 进度条 80%（正常色）；当月支出 1200 → 进度条超出（红色）
- [ ] **AC-2.4** 搜索：输入"午餐" → 列表实时过滤，只显示备注包含"午餐"的记录
- [ ] **AC-2.5** CSV 导出：点击导出按钮 → 下载 CSV 文件，用文本编辑器打开验证内容正确
- [ ] **AC-2.6** CSV 导入：准备一个 CSV 文件 → 导入 → 显示预览 → 确认 → 数据出现在列表中
- [ ] **AC-2.7** 主题切换：点击切换按钮 → 页面切换到深色模式 → 刷新页面后仍保持深色模式
- [ ] **AC-2.8** 无数据图表：当月无支出时，饼图和柱状图显示空状态提示而非空白

### 数据结构

```javascript
// 单条支出记录
{
  id: "uuid-string",       // 唯一标识
  amount: 100,             // 金额（正数）
  category: "餐饮",        // 分类名称
  date: "2026-03-15",      // 日期 YYYY-MM-DD
  note: "午餐"             // 备注
}

// localStorage key: "expenses" → JSON.stringify([...records])
// localStorage key: "budget" → JSON.stringify({ monthly: 1000 })  // M2
// localStorage key: "theme" → "light" | "dark"                    // M2
```
