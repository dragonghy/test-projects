# Interactive Quiz Builder - 互动测验应用

**在线演示**: https://quiz-app-dun-psi.vercel.app

**部署状态**: ✅ 已部署到 Vercel

## 产品概述

### 产品目标
开发一个纯前端互动测验应用，让用户能够轻松创建和分享趣味测验。应用无需后端，通过 JSON 文件实现测验的导入导出和分享。

### 目标用户
- 教师：创建课堂测验检验学生学习效果
- 培训师：制作培训后的知识检测
- 普通用户：制作趣味测验分享给朋友

### 技术约束
- 纯前端：HTML + CSS + JavaScript（无框架）
- 部署方式：Vercel 静态部署
- 代码目录：`projects/quiz-app/`
- 文件结构：`index.html`, `style.css`, `app.js`

---

## 核心体验

应用有两种模式：

1. **创建模式**：用户编辑测验标题、添加多道选择题（每题 2-4 个选项，标记正确答案），完成后导出为 JSON 分享
2. **答题模式**：加载一个测验后逐题作答，有倒计时，最后显示得分和详细结果

## Milestone 规划

### M1：基础功能（核心 MVP）

#### 功能清单

**F1 - 主页**
- 显示应用标题 "Quiz Builder"
- 两个入口按钮：「创建测验」和「开始答题」
- 现代 UI，渐变主题色背景

**F2 - 创建模式**
- 输入测验标题（必填，placeholder 提示）
- 添加题目：点击「添加题目」按钮新增一道题
- 每道题包含：
  - 题干输入框（必填）
  - 2-4 个选项输入框（至少 2 个必填）
  - 「添加选项」按钮（最多 4 个选项时隐藏）
  - 每个选项旁有单选按钮标记正确答案（必选一个）
  - 「删除题目」按钮
- 实时显示题目数量（如 "已添加 3 道题"）
- 至少添加 1 道题且标题不为空才能导出

**F3 - 导出功能**
- 「导出 JSON」按钮，将测验数据下载为 `.json` 文件
- 文件名格式：`quiz-<标题>.json`
- JSON 结构：
```json
{
  "title": "测验标题",
  "questions": [
    {
      "question": "题干文字",
      "options": ["选项A", "选项B", "选项C", "选项D"],
      "correctIndex": 0
    }
  ]
}
```

**F4 - 导入功能**
- 「开始答题」页面提供「导入 JSON」按钮
- 支持通过文件选择器选择 `.json` 文件
- 导入后校验格式，格式错误时显示友好提示
- 同时提供「使用示例测验」按钮，加载内置示例

**F5 - 答题模式**
- 逐题显示：题号（如 "第 1/5 题"）、题干、选项按钮
- 点击选项后立即反馈：
  - 正确：选项变绿色 + 显示对勾
  - 错误：选中的选项变红色 + 显示叉号，正确选项变绿色
- 反馈显示 1 秒后自动跳转到下一题
- 最后一题回答后跳转到结果页面

**F6 - 结果页面**
- 显示总分：正确数 / 总题数
- 显示百分比分数
- 每道题的答题详情：题干、用户选择、正确答案、是否正确
- 「重新答题」按钮（重新开始同一测验）
- 「返回主页」按钮

**F7 - 草稿保存**
- 创建模式中的数据自动保存到 localStorage
- 下次进入创建模式时自动恢复草稿
- 提供「清空草稿」按钮

**F8 - 响应式设计**
- 桌面端：居中卡片布局，最大宽度 640px
- 移动端：全宽布局，适当内边距
- 按钮和输入框有足够的触摸区域（最小 44px 高度）

**F9 - 视觉设计**
- 现代渐变主题色（推荐蓝紫渐变）
- 圆角卡片、阴影效果
- 平滑过渡动画（页面切换、反馈显示）
- 清晰的字体层级

### M2：增强功能

#### 功能清单

**F10 - 倒计时**
- 创建测验时可设置每题时间限制：10/20/30 秒，或不限时
- 答题时顶部显示倒计时进度条（视觉化）
- 超时自动判为错误并跳到下一题
- 超时时显示 "时间到！" 提示

**F11 - 答题进度条**
- 答题页面顶部显示进度条
- 格式：进度条 + 文字 "第 X/Y 题"
- 进度条随答题推进填充

**F12 - 分数等级**
- 结果页面根据百分比显示等级：
  - 90-100%: A 级
  - 80-89%: B 级
  - 70-79%: C 级
  - 60-69%: D 级
  - 0-59%: F 级
- 每个等级配对应 emoji（如 A=🏆, B=🎉, C=👍, D=😅, F=💪）

**F13 - 选项乱序**
- 每次加载答题时，每道题的选项顺序随机打乱
- 打乱后 correctIndex 相应更新，确保答案判定正确

**F14 - 答题历史**
- localStorage 保存最近 10 次答题结果
- 主页新增「答题历史」入口
- 历史列表显示：测验标题、得分、日期时间
- 点击可查看该次答题的详细结果

**F15 - 快捷键**
- 答题时按数字键 1-4 选择对应选项
- 结果页按 Enter 重新答题
- 任意页面按 Esc 返回主页
- 首次进入答题模式时显示快捷键提示

**F16 - 深色/浅色主题**
- 主页或导航栏提供主题切换按钮（太阳/月亮图标）
- 深色主题：深色背景、浅色文字、调整卡片和按钮颜色
- 主题偏好保存到 localStorage
- 默认跟随系统偏好（`prefers-color-scheme`）

| AC11 | 创建测验时可设置倒计时（10/20/30秒/不限时） | 创建带不同时间限制的测验 |
| AC12 | 答题时倒计时正确显示和运行，超时自动跳题 | 等待倒计时结束验证 |

## JSON 数据结构

### 测验数据（M1）
```json
{
  "title": "示例测验",
  "questions": [
    {
      "question": "HTML 的全称是什么？",
      "options": [
        "Hyper Text Markup Language",
        "High Tech Modern Language",
        "Hyper Transfer Markup Language",
        "Home Tool Markup Language"
      ],
      "correctIndex": 0
    }
  ]
}
```

### 测验数据（M2 扩展，含倒计时）
```json
{
  "title": "示例测验",
  "timeLimit": 20,
  "questions": [...]
}
```

### 答题历史记录
```json
{
  "quizTitle": "示例测验",
  "score": 4,
  "total": 5,
  "percentage": 80,
  "grade": "B",
  "date": "2026-03-16T10:30:00Z",
  "details": [
    {
      "question": "题干",
      "userAnswer": 1,
      "correctAnswer": 0,
      "isCorrect": false
    }
  ]
}
```

## 设计规范

- **主色调**：蓝紫渐变 (`#667eea` → `#764ba2`)
- **圆角**：按钮 8px，卡片 12px
- **阴影**：卡片 `0 4px 6px rgba(0,0,0,0.1)`
- **字体**：系统字体栈 `-apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif`
- **动画**：过渡效果 0.3s ease
